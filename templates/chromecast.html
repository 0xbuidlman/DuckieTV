ChromeCast support (not finished)

Hi there!

As a proof-of-concept, I have included this experimental ChromeCast support in this build. It's not done yet, so use at your own risk and don't come knocking at my door if your computer or ChromeCast dongle implodes.

Requirements
  • A ChromeCast dongle (which you already have since you see this page)
  • µTorrent 3.3.x on Windows (3.4.x will not work, yet, downgrade if you want to experiment with this feature)
  • µTorrent webUI enabled and configured for DuckieTV. For more info on how to set this up:

    Click the 'Enable Web UI' checkbox in Preferences -> Advanced -> Web UI. Set the authentication to admin/admin (this will be configurable in the future) I strongly advise only to allow access from 127.0.0.1 and your ChromeCast's IP.

  • Your local IP Address selected below

The way I got Chromecast support to work is by piggy-backing on µTorrent's 'stream' function. This is awesome because you can push a video directly from your local computer without DuckieTV having to worry about transcoding or anything.

The downside to this is that µTorrent embeds '127.0.0.1' as the default target for the streaming URL (On top of requiring HTTP authentication).

To make sure I can open up a connection from your chromecast dongle to this machine, you need to click the button below to select your local machine's IP. If you have more than one IP address, please select the one belonging to your WIFI connection.

Found Local ip address

ChromeCast Local Streaming IP Address: {{localIpAddress}}

You're all set now! Press Connect to start Casting!

After you've connected you can Go to the torrent remote page and press the 'Play on Chromecast' button on any supported fileformat to stream it!

Notes:

A lot of the required UI for ChromeCast compliance is not working. You can only start a video, not stop it, you can't control the volume, you can't resume your session. This will all be finished in v0.6! (And any help would be more than welcome!)
